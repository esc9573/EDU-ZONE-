1. Introduction

1.1 Introduction to C#

We are going to learn a language called C# (pronounced C sharp). If you ever make the mistake of calling the language C hash you will show your ignorance straight away! C# is a very flexible and powerful programming language with an interesting history. It was developed by Microsoft Corporation for a variety of reasons, some technical, some political and others marketing. C# bears a strong resemblance to the C++ and Java programming languages, having borrowed (or improved) features provided by these languages. The origins of both Java and C++ can be traced back to a language called C, which is a highly dangerous and entertaining language which was invented in the early 1970s. C is famous as the language the UNIX operating system was written in, and was specially designed for this.
Definition of C#
C# (pronounced "C Sharp") is a modern, high-level programming language developed by Microsoft in 2000. It is an object-oriented, strongly typed language designed for building a wide range of applications, including desktop, web, mobile, and gaming applications. C# is part of the .NET ecosystem and is widely used for developing Windows applications, cloud services, and backend systems.

1.2 Key Features of C#

C# offers several powerful features, such as: Object-Oriented Programming (OOP): Supports classes, inheritance, polymorphism, and encapsulation.
Automatic Memory Management: Uses garbage collection to handle memory allocation and deallocation. Type Safety: Prevents type-related errors at compile time. Asynchronous Programming: Supports async and await for efficient multitasking. Cross-Platform Development: Works on Windows, Linux, and macOS via .NET Core.

1.3 What Comprises a C# Program?

If your mum wanted to tell you how to make your favourite fruitcake she’d write the recipe down on a piece of paper. The recipe would be a list of ingredients followed by a sequence of actions to perform on them. A program can be regarded as a recipe, but written for a computer to follow, not a cook. The ingredients will be values (called variables) that you want your program to work with. The program itself will be a sequence of actions (called statements) that are to be followed by the computer.
Rather than writing the program down on a piece of paper you instead put it into a file on the computer, often called a source file. This is what the compiler acts on. A source file contains three things:
1- instructions to the compiler
2- information about the structures which will hold the data to be stored and manipulated.
3- instructions which manipulate the data.

2.2 Switch Statement (Multi-Case Selection)

We now know nearly everything you need to know about constructing a program in the C# language.
You may find it rather surprising, but there is really very little left to know about programming itself. Most of the rest of C# is concerned with making the business of programming simpler. A good example of this is the switch construction.
How does a switch statement simplify multiple conditions?
Answer: The switch statement selects one of many code blocks to execute based on a value.

string day = "Monday";
switch (day)
{
case "Monday":
Console.WriteLine("Start of the week!");
break;
case "Friday":
Console.WriteLine("Weekend is near!");
break;
default:
Console.WriteLine("Another day.");
break;
}More efficient than multiple if-else statements.
Requires break to exit each case.

using System;

3. The Basic In C#

3.1 What is a Function in C#?

In C#, functions are more commonly called methods, and they are one of the most important building blocks in the language. They allow you to group lines of code into one reusable block that you can run (or "call") whenever you need it. Imagine you’re baking a cake . Instead of repeating all the steps every time you bake, you write a recipe. That recipe is like a function. You call it whenever you need it. In C#, a function lives inside a class, and it can perform actions, return values, or do both.
returnType FunctionName(parameters)
{
// code to execute
return value; // if returnType is not void
}

3.2 Objects and Structures

In C# objects and structures have a lot in common. They can both hold data and contain methods. However, there is a crucial difference between the two. Structures are managed in terms of value whereas objects are managed in terms of reference. It is very important that you understand the distinction between the two, for it has a big impact on the way that they are used.

3.3 Why Use Functions?

Using functions is not just good programming practice, it's essential. Here’s why:
1. Reusability If you need to do the same task multiple times, you don’t want to copy and paste the same code everywhere. Just write a function once, and call it whenever you need it.
2. Modularity Functions help break your program into small, understandable pieces. Instead of one long piece of code that does everything, you have smaller blocks that each do one thing. This makes the code easier to read, test, and debug.
3. Maintainability If a bug appears in a calculation that’s used many times, you only need to fix the bug inside the function. All parts of your program that use that function will now use the corrected version.
4. Clarity Naming a function gives meaning to what the code does. For example: CalculateSalary() is much clearer than just a chunk of math operations sitting in your Main().

3.4 Anatomy of a Function in C#

Let’s break down what a function is made of in C#:

return_type FunctionName(parameter1Type param1, parameter2Type param2, ...)
{
// Function body (statements)
return value; // if not void
}

Part Meaning
return_type The type of value the function will return (e.g., int, string, void if nothing is returned)
FunctionName The name you give to the function so you can call it later.
parameters Inputs (arguments) you pass into the function so it can work with different values.
return The keyword used to send a result back to the caller. Not needed if return type is void.

int Multiply(int x, int y)
{ return x * y; }

3.5 Calling a Function

You don’t just write a function — you need to call it to use it.

int result = Multiply(3, 4);
Console.WriteLine(result); // Output: 12 When you call Multiply(3, 4), it executes the code inside and gives you the result 12.



3.6 Types of Functions

In C#, there are several types of functions based on how you define and use them.
1. Functions with Parameters and Return Value
These take input and return output.

int Add(int a, int b)
{
return a + b;
} Used when you want to calculate and return a value.

Functions with Parameters and No Return Value
These take input, do something (like print), but don’t return anything.

void PrintName(string name)
{
Console.WriteLine("Your name is " + name);
} Used for displaying messages, logging, or any action that doesn’t produce a value.

3.7 Functions with No Parameters and No Return

Just performs a fixed task when called.

void SayHello()
{
Console.WriteLine("Hello!");
} Used when the action is static and doesn’t depend on input.

3.8 Static vs Instance Functions

Static Functions Belong to the class, not to any specific object.
Called using the class name.
Often used in Main() or helper utilities.

static void Greet() {
Console.WriteLine("Hi there!");
} Call with: ClassName.Greet();

3.9 Arguments and Parameters

When you read about C# you will find the words parameter and argument used interchangeably. This is actually not quite right. Understanding the difference will make C# documentation and error messages seem more sensible and might even give you a feeling of superiority over lessor programmers who don’t know the truth. A parameter is the special kind of variable that is defined in the method header and used inside that method to represent the value that was fed into the method call.

static int sillyReturnPlus ( int i)
{
i = i + 1;
Console.WriteLine ( "i is : " + i ) ;
return i;
}The method sillyReturnPlus above has a single parameter, which is of type integer and has the identifier i.
